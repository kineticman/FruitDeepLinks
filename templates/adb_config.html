Provider Lane Configuration

What is this? ADB lanes allow integration with ADBTuner and ChromeCapture for Channels DVR. Each enabled provider gets dedicated virtual channels that can be tuned via ADB commands.

How it works:

  • Enable providers you want to expose via ADB
  • Set the maximum lanes per provider — actual channel count equals peak concurrent events (up to this limit)
  • Export generates M3U/XMLTV files for ADBTuner integration
  • Changes here do NOT affect your normal Channels DVR view

After saving changes, run "Apply Filters Now" from the Filters page or trigger a refresh to regenerate ADB lane files.

Device Profiles

Each ADB lanes M3U is available in two device profiles that control which deep link style is embedded. Use the /m3u/adb endpoint and pass a profile query parameter to select the style your client expects.

Profile Endpoint Link style Best for
fire (default) /m3u/adb or /m3u/adb?profile=fire aiv://, sportscenter://, … (native app scheme) Fire TV, Android (ADBTuner)
apple /m3u/adb?profile=apple https://app.primevideo.com/…, https://www.espn.com/watch/…, … (HTTPS) Apple TV, iOS, web clients (CC4C, PrismCast)

Per-provider files follow the same convention — the apple variant adds _apple before the extension:

  • /out/adb_lanes_aiv.m3u → Fire TV / Android
  • /out/adb_lanes_aiv_apple.m3u → Apple TV / HTTPS

Quick Guide

Example Configuration:

  • ESPN+: Enable, 5 lanes → Creates channels for ESPN+ events 1–5
  • Peacock: Enable, 3 lanes → Creates channels for Peacock events 1–3
  • Paramount+: Disable → No ADB channels created

Recommended Lane Counts:

  • Small providers (1–10 events): 1–2 lanes
  • Medium providers (10–50 events): 3–5 lanes
  • Large providers (50+ events): 5–10 lanes
